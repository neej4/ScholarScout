# Master's Research Projects

## Profile
- **Duration:** 6-12 months
- **Compute:** Cloud GPU access (A100/L4, ~$50-200 total)
- **Budget:** Low to moderate (cloud credits, API costs)
- **Scope:** Novel combination of methods OR new application of existing technique

## What Makes a Good Master's Project
- Addresses a specific gap identified in recent literature
- Combines 2-3 existing techniques in a novel way
- Requires non-trivial engineering (custom training loop, data pipeline)
- Produces publishable results (workshop paper or short paper)

## Typical Structure
1. Deep literature survey (1-2 months)
2. Design methodology and architecture (1 month)
3. Implement and iterate (2-3 months)
4. Experiments and ablation studies (1-2 months)
5. Write thesis/paper (1-2 months)

## Resource Constraints
- Model size: up to ~7B parameters (with LoRA/QLoRA)
- Dataset: up to ~100GB
- Training: 1-5 days on single A100
- May use APIs (OpenAI, HuggingFace) with budget limits
