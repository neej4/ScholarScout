# Hackathon Project

## Profile
- **Duration:** 4-12 hours (single sprint)
- **Compute:** Laptop + free APIs only
- **Budget:** $0 (everything must be free tier)
- **Scope:** Working demo that can be presented in 3-5 minutes

## Constraints
- Must be demo-able at the end of the sprint
- No training ML models from scratch (use pre-trained or APIs)
- Must have visible UI or clear output (judges need to see something)
- Prefer integration over invention (combine existing tools in novel ways)
- Code quality matters less than working prototype
- Must be explainable to non-technical judges in 2 minutes

## Output Expectations
- Working prototype (not slides, not mockups)
- 3-minute live demo
- Clear problem statement + solution in one sentence
- Measurable impact claim (time saved, errors reduced, etc.)

## What Works in Hackathons
- Plugins/extensions for existing platforms
- Workflow automation that saves measurable time
- Clever combination of 2-3 APIs
- Tools that solve a specific pain point for a specific user

## What Doesn't Work
- Research papers disguised as projects
- ML training pipelines (too slow to demo)
- Overly complex architectures
- Solutions without clear users
